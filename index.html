GIT CONFIG

O comando git config é usado para configurar as opções do Git. Ele pode ser usado para definir opções como nome de usuário, endereço de e mail, editor preferido, entre outras. Existem três níveis de configuração: 1.Configuração global: Aplica-se a todos os repositórios do usuário em um determinado sistema. Para definir a configuração global, use o comando:

git config --global <opção> <valor>

Por exemplo, para definir o nome de usuário globalmente, use:

git config --global user.name "Seu Nome"

2. Configuração local do repositório: Aplica-se apenas ao repositório atual em que o comando é executado. Para definir a configuração local, use o comando:

git config <opção> <valor>

Por exemplo, para definir o nome de usuário localmente, use:

git config user.name "Seu Nome"

3. Configuração específica do arquivo: Aplica-se apenas ao arquivo de configuração do Git específico que está sendo usado. Para definir a configuração específica do arquivo, use o comando:

git config --file <arquivo-de-configuração> <opção> <valor>

Por exemplo, para definir o nome de usuário em um arquivo de configuração específico, use:

git config --file /caminho/para/arquivo-de- configuração user.name "Seu Nome"

Alguns exemplos de opções do Git que podem ser configuradas usando o git config são:

  • user.name: Define o nome de usuário.
  • user.email: Define o endereço de e-mail.
  • core.editor: Define o editor de texto preferido para o Git.
  • color.ui: Habilita ou desabilita a saída colorida do Git.
  • merge.tool: Define a ferramenta de mesclagem de conflitos preferida.
  • push.default: Define a estratégia padrão para empurrar alterações para um repositório remoto.
